# UI Field Schema для World Edit

## Назначение
Схема описывает live inline-поля генераторов для `WorldEditPanel`. Это единственный пользовательский путь настройки параметров для готовых генераторов.

## Базовый объект поля
```dm
list(
    "id" = "field_id",
    "label" = "Название поля",
    "kind" = "select|number|boolean|text",
    "value" = <текущее значение>
)
```

## Обязательные ключи
1. `id` - строковый идентификатор параметра.
2. `label` - отображаемое название поля.
3. `kind` - тип виджета.
4. `value` - текущее значение.

## Поддерживаемые `kind`
1. `select`
2. `number`
3. `boolean`
4. `text`

## Optional ключи
1. `options` - только для `select`.
2. `min`, `max`, `step` - только для `number`.
3. `description` - короткое описание поля.
4. `validate_hint` - подсказка по ограничениям.
5. `placeholder` - placeholder для `text`.
6. `group` - логическая группа полей в UI.
7. `visible` - скрыть поле при `FALSE`.
8. `disabled` - сделать поле read-only.
9. `required` - визуальная маркировка обязательного поля.

## Формат `options`
```dm
list(
    list("label" = "Вариант A", "value" = "a", "description" = "Комментарий"),
    list("label" = "Вариант B", "value" = "b")
)
```

## Правила backend-нормализации
1. Поля без `id` отбрасываются.
2. Поля с `visible=FALSE` отбрасываются.
3. Поля с неподдерживаемым `kind` отбрасываются.
4. Для `select` options нормализуются, label-коллизии устраняются.
5. При отсутствии `group` используется `Основные`.

## Правила `set_ui_param`
1. Возврат `list` - успешное обновление параметров.
2. Возврат `string` - пользовательская ошибка, показывается в UI и `to_chat`.
3. Возврат `null` - операция не изменила состояние.

## Live-only правила
1. Если генератор возвращает пустой набор полей, UI не подставляет другой экран настройки.
2. `refresh_ui_state` используется для динамических каталогов и пересборки `ui_fields`.
3. Валидация и фильтрация выполняются backend-ом, а не отдельным запасным экраном.

## Рекомендации по качеству
1. Использовать `group` для длинных форм.
2. Для каждого числового лимита задавать `validate_hint`.
3. Для динамических каталогов обновлять поля через `refresh_ui_state`.
4. Для условных полей использовать `disabled` или `visible`.
